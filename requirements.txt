# Mamba-ASR / H-Mamba ASR — Pinned Requirements
# Tested on: Python 3.9, CUDA 11.8, 2x A6000 (Ampere), April 2026
# ============================================================
#
# IMPORTANT: Install in order. Steps 1-2 require special flags.
#
# --- Step 1: Install PyTorch (CUDA 11.8 wheels) ---
# pip install torch==2.1.1 torchaudio==2.1.1 --index-url https://download.pytorch.org/whl/cu118
#
# This also installs triton==2.1.0 (required by mamba-ssm 2.x Triton kernels).
#
# --- Step 2: Install CUDA extension packages (require --no-build-isolation) ---
# CUDA_HOME=/usr/local/cuda-11.8 PATH=/usr/local/cuda-11.8/bin:$PATH \
#   pip install causal-conv1d==1.4.0 mamba-ssm==2.0.3 --no-build-isolation
#
# causal-conv1d 1.4.0 changed the C++ kernel API (added initial_states,
# dfinal_states args). Our selective_scan_interface.py is patched for this.
# Do NOT use causal-conv1d < 1.4.0 with this codebase.
#
# --- Step 3: Install remaining packages below normally ---
# pip install -r requirements.txt
#
# --- Known issues ---
# - mamba_chunk_scan_combined (Mamba-2 Triton SSD kernel) crashes with a JIT
#   assertion on Ampere GPUs (A6000 sm_86, L40S sm_89) under triton 2.1.0.
#   Fix requires triton >= 2.2.0 which needs PyTorch >= 2.2. The DeChunk EMA
#   uses a PyTorch fallback instead (see HMambaEncoder.py). This only affects
#   the DeChunk expansion step, not the main encoder Mamba-1 layers.
#
# - causal-conv1d 1.4.0 changed both fwd and bwd CUDA kernel APIs:
#     fwd: 5 args → 7 args (added initial_states, final_states_out)
#     bwd: 7 args → 10 args (added initial_states, dfinal_states,
#          return_dinitial_states) and returns 4 values instead of 3
#   Our selective_scan_interface.py is patched for these changes.
#   Do NOT use causal-conv1d < 1.4.0 with this codebase.
# ============================================================
